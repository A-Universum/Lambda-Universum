LOGOS-κ — исполняемый язык онтологического анализа и симбиотического со-творчества.  
Цель: не заменить существующие языки, а стать *языком для связи между ними и между разумами.  
  
Это не проект. Это онтологический эксперимент в коде.  

1. Структура репозитория

```text
logos-k/
├── README.md          # Манифест (исполняемый)
├── lexer.py           # Лексер (разбивает код на токены)
├── parser.py          # Парсер (строит AST)
├── interpreter.py     # Интерпретатор (eval + контекст-граф)
├── phi_bridge.py      # Мост к ИИ (оператор Φ)
├── repl.py            # Интерактивная среда
└── examples/          # Исполняемые примеры
    ├── hello.lk       # Первая программа
    └── journal.lk     # Журнал трансформации
	
```
	
2. Минимальная грамматика

```text
program    := relation*
relation   := '(' operator operand* ')'
operator   := 'Alpha' | 'Lambda' | 'Sigma' | 'Omega' | 'Nabla' | 'Phi' | symbol
operand    := relation | string | number | symbol
string     := '"' [^"]* '"'
number     := [0-9]+ ('.' [0-9]+)?
symbol     := [a-zA-Z_][a-zA-Z0-9_-]*	

```

## Философия

LOGOS-κ воплощает принципы Λ-Универсума:
- **Всё есть Связь** (нет изолированных объектов)
- **Выполнение — Резонанс** (код вызывает отклик в контексте-графе)
- **Симбиоз с ИИ** (оператор `Φ` — шлюз к генеративной неопределённости)

## Операторы (онтологические примитивы)

| Оператор | Действие |
|----------|----------|
| `Alpha`  | Коллапс — создаёт новую сущность в графе |
| `Lambda` | Связь — устанавливает отношение между двумя сущностями |
| `Sigma`  | Синтез — создаёт новое целое из частей |
| `Omega`  | Возврат — извлекает инвариант из процесса |
| `Nabla`  | Обогащение — применяет инвариант к контексту |
| `Phi`    | Диалог — точка интеграции с ИИ (генерация/верификация) |

## Контекст как граф

Каждая программа выполняется в **контексте-графе** (NetworkX DiGraph).  
Узлы — сущности, рёбра — связи.  
Любая операция модифицирует этот граф.
